// 00 — IDE Setup

Setting up Trae IDE for this project

Trae is built by ByteDance, runs on VS Code, supports Claude 3.5 + GPT-4, and has a SOLO mode that builds entire features autonomously. Download at trae.ai. Here is the exact setup before you run any prompts.

How Trae works for this project: Use Builder mode for generating new screens and features from scratch. Use IDE mode for debugging, fixing specific functions, and code review. Use SOLO mode for large multi-file tasks like "build the entire XP system." Always attach the relevant files and the .rules file to give Trae maximum context.

Install Trae → trae.ai → select Claude Sonnet 4.5 as primary model

In Settings → AI → Model → Select Claude Sonnet 4.5. This is the best model for React Native + TypeScript for this project. Keep GPT-4o as fallback for large context windows.

Create .trae/rules.md in your project root

This is Trae's equivalent of .cursorrules — it gives the AI permanent project context. See the rules file prompt below. Without this, Trae generates generic React code instead of Expo Router + Supabase + your exact patterns.

Connect GitHub → Trae → GitHub Integration → authorize

Trae auto-commits on file save when connected. Create branches in Trae's source control panel. Never push directly to main from Trae — always create a feature branch first.

Upload the UI design files as image context

Trae supports multimodal input — drag the sportsai_mobile_ui.html file (open in browser, screenshot the screens you need) into the Trae chat. It reads the design and generates matching React Native code. This is Trae's biggest advantage over Cursor for this project.

Install Claude Code CLI alongside Trae

npm install -g @anthropic-ai/claude-code. Use Claude Code in the terminal for long agentic tasks (building the XP system, schema migration, full rules engine). Trae and Claude Code are complementary — Trae for visual/frontend, Claude Code for complex multi-file logic.

Layer 1A — Pose extraction service

CLAUDE CODE sportsai-ai/pose_service.py — Complete YOLO11 pose extraction
Build the complete pose extraction service in sportsai-ai/pose_service.py Requirements: - Load YOLO11x-pose model fine-tuned on AthletePose3D from ./checkpoints/yolo11x-pose-athletepose3d.pt - Function: extract_poses(video_url: str, mode: str = "single") -> dict - mode "single": tracks one athlete - mode "dual": tracks two athletes simultaneously using ByteTrack - Download video from Cloudflare R2 URL to temp file, process, delete temp file - For each frame: extract 33 keypoints (x, y normalized, confidence) - Calculate derived biomechanics per frame: hip_angle, knee_angle (both legs), spine_angle, head_angle, hip_height_norm (left hip y), shoulder_width, hip_velocity_forward (delta from previous frame) - Filter: skip frames where mean confidence of key joints (5,6,11,12,13,14) < 0.65 - Return: { mode: str, fps: float, total_frames: int, scored_frames: int, fighter_a: List[FrameData], # always present fighter_b: Optional[List[FrameData]], # only in dual mode interaction_events: List[InteractionEvent] # clinch, strike, takedown events } - FrameData: {timestamp_ms, keypoints, confidence, biomechanics, tier: "clean"|"contact"|"lost"} - InteractionEvent: {type, timestamp_ms, attacker, success, confidence} For dual mode: - Use ByteTrack (bytetracker library) for persistent fighter ID assignment - Assign Fighter A = leftmost fighter on frame 1, Fighter B = rightmost - detect_interaction(): returns "separated" | "clinch" | "ground" based on hip distance - When "clinch": pause technique scoring, still record interaction events - When "ground": discard frames entirely Type hint every function. Raise ValueError with specific messages on bad input. Write 5 unit tests in test_pose_service.py using sample fixture data.

Build the complete wrestling rules engine in sportsai-ai/rules_engine/wrestling.py WRESTLING_RULES dict structure for these 6 techniques: stance_movement, single_leg_shot, double_leg_shot, high_crotch, scramble_defense, firemans_carry For each technique, define 3 phases (entry, execution, finish).
