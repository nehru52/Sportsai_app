// Key architecture decisions for the mobile app

State management: Zustand + React Query

Zustand for UI state (current screen, user session, active fight data). React Query for all server state (analyses, leaderboards, XP). Never put server data in Zustand. Never put UI state in React Query. This split prevents 80% of state bugs.

Navigation: Expo Router (file-based)

Expo Router uses file-based routing like Next.js. app/(tabs)/home.tsx = the home tab. app/analysis/[id].tsx = analysis detail screen. No manual navigation config. Cursor understands this pattern well — just describe the screen you want and it creates the file.

Rendering: Skia for 2D, Three.js for 3D

Skia canvas handles the skeleton overlay on the annotation player (2D, fast, precise). Three.js handles the avatar rendering (3D, GLB loading, particle effects). Never use both on the same screen — too much GPU load. Annotation player = Skia. Avatar viewer = Three.js.

Video: Expo Video + Cloudflare Stream

Expo Video plays HLS streams from Cloudflare Stream (adaptive bitrate, thumbnails, scrubbing). The annotation player positions the Skia canvas as an absolute overlay on top of the Expo Video component using React Native's absolute positioning system.

Anime fight cinematic system

The complete technical setup for the anime fight mode. Three parts: DomoAI style transfer API, Three.js power effects, and the compositing pipeline.

// anime_service/domo_client.py 
import httpx
import asyncio

DOMO_API_BASE = "https://api.domoai.app/v1"

async def convert_to_anime(video_url: str, style: str = "dragon_ball_z") -> str:
    """
    Convert fight clip to anime style via DomoAI API.
    Returns URL of anime-styled video.
    Cost: ~$0.10-0.30 per 30-second clip.
    """
    async with httpx.AsyncClient() as client:
        # Submit job
        response = await client.post(
            f"{DOMO_API_BASE}/video2video",
            headers={"Authorization": f"Bearer {DOMO_API_KEY}"},
            json={
                "video_url": video_url,
                "style": style,         # "dragon_ball_z", "naruto", "custom"
                "strength": 0.75,       # Style transfer strength (0.0-1.0)
                "preserve_motion": True # Keep original movement timing
            }
        )
        job = response.json()
        job_id = job["job_id"]

        # Poll for completion (30-120 seconds)
        while True:
            status = await client.get(
                f"{DOMO_API_BASE}/jobs/{job_id}",
                headers={"Authorization": f"Bearer {DOMO_API_KEY}"}
            )
            result = status.json()
            if result["status"] == "completed":
                return result["output_url"]
            await asyncio.sleep(5)
// mobile/components/PowerEffects.tsx — Three.js power aura system 
import { useFrame } from '@react-three/fiber'
import { useRef } from 'react'
import * as THREE from 'three'

// Aura particle system — driven by real technique scores
export function PowerAura({ techniqueScore, errorActive, position }) {
  const particlesRef = useRef()

  // Aura color = function of real biomechanical score
  const auraColor = techniqueScore >= 85 ? '#FFD700'  // Gold — mastery
    : techniqueScore >= 70 ? '#4488FF'               // Blue — technical
    : errorActive ? '#FF4466'                        // Red — error
    : '#FFFFFF'                                        // White — neutral

  const auraIntensity = techniqueScore / 100  // Score directly drives glow

  useFrame((state) => {
    if (!particlesRef.current) return
    // Animate particles — faster when score is higher
    particlesRef.current.rotation.y += 0.01 * auraIntensity
    // Flicker on error
    if (errorActive) {
      particlesRef.current.material.opacity = 0.4 + Math.sin(state.clock.elapsedTime * 8) * 0.3
    }
  })

  return (
    <points ref={particlesRef} position={position}>
      <sphereGeometry args={[0.8, 32, 32]} />
      <pointsMaterial
        color={auraColor}
        size={0.03}
        transparent
        opacity={auraIntensity * 0.7}
        sizeAttenuation
      />
    </points>
  )
}

// 14 — Deployment pipeline

How code goes from laptop to production

API (Railway) — automatic

Push to main → Railway detects the push → builds Docker image → deploys → health check. Zero manual steps. Takes 2–3 minutes. If health check fails, Railway auto-rolls back to previous deployment. Set this up on day 1 and never think about it again.

Coach Web Dashboard (Vercel) — automatic

Push to main → Vercel builds Next.js → deploys to global CDN. Preview deployments on every PR (dev branch gets a unique preview URL). Zero config after initial setup.

AI Service (Modal) — on-demand

Modal deploys on push via: modal deploy modal_app.py — add this to GitHub Actions. The GPU function scales to zero when not in use (no idle cost) and cold-starts in ~3 seconds when called.

Mobile App (Expo EAS) — manual or OTA

Bug fixes: eas update — pushes OTA update, no App Store review needed. New native features: eas build && eas submit — triggers TestFlight/Play Store submission. OTA updates are instant. Native builds take 15–30 minutes.
